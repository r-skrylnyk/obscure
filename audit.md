## Призначення

CLI-інструмент, що обфускує назви CSS-класів та ID у наборі `.css` і `.html` (або `.php`) файлів. Генерує перейменовані файли та JSON-карту відповідностей оригінал → обфускована назва. Без мініфікації, без бандлінгу — виключно перейменування селекторів.

## CLI-інтерфейс

```
obscure <include> [options]

  <include>               Список .css файлів через кому або glob-шаблон,
                          з яких беруться визначення класів/ID для обфускації

Параметри:
  -a, --apply <apply>     .html/.php файл(и), до яких застосовується обфускація
  -e, --exclude <exclude> .css файл(и), чиї визначення НЕ треба обфускувати
  -o, --output <output>   Директорія виводу (за замовчуванням: поточна директорія)
  -s, --seed <seed>       Ціле число-сід для відтворюваної обфускації
  -V, --version           Виводить 1.0.1
```

## Внутрішня архітектура

### Потік виконання

```
/usr/local/bin/obscure → ../lib/node_modules/obscure/index.js  (npm symlink)
  └─ index.js: commander парсить argv → викликає obscure(opts)
```

Поле `bin` в `package.json` змінено з `./bin/obscure.js` на `./index.js`, що відповідає фактичному симлінку, який npm створює під час глобальної установки (підтверджено логами Docker).

`bin/obscure.js` більше не використовується як точка входу і залишений для сумісності.

---

### Покроковий процес виконання (функція `obscure()` в `index.js`)

#### 1. Побудова мапи виключень (`--exclude`)

Зчитує кожен виключений `.css` файл через `getDefinitions()` та додає всі знайдені селектори до масиву `excluded[]`. Ці селектори пропускаються під час побудови мапи включень.

#### 2. Побудова мапи включень (`<include>`)

Для кожного включеного `.css` файлу:
- Зчитує вміст файлу
- Запускає `getDefinitions()` для отримання всіх селекторів класів/ID
- Для кожного селектора, якого **немає** у списку виключень, генерує обфусковану назву:
  ```js
  hashids.encode(i + opts.seed)
  // i = індекс селектора у списку визначень поточного файлу
  // opts.seed = Date.now() при запуску АБО ціле число з --seed
  ```
- Зберігає сирий рядок CSS у `csss{}` для подальшої заміни тексту
- Зберігає `{ sym, origin, obfused }` у `map{}` з ключем — рядком селектора

> **Примітка**: hashids v2 використовує інший алгоритм кодування, ніж v1. При однаковому сіді результат буде **інший**, ніж у v1. Якщо ви зберігаєте `obscure.map` і порівнюєте між збірками, переконайтесь що скрізь використовується однакова версія.

#### 3. Побудова DOM-мапи (`--apply`)

Для кожного файлу з `--apply` зчитує HTML у об'єкт DOM cheerio. Усі DOM-об'єкти зберігаються в пам'яті у `doms{}`.

#### 4. Прохід обфускації

Для кожного запису у `map`:
- **CSS-файли**: пошук і заміна регулярним виразом у сирому рядку:
  ```js
  new RegExp("\\" + sym + origin + "(?=[#.,\{\[\(\s])", "g")
  ```
  Lookahead гарантує заміну лише у позиціях селектора (а не, наприклад, у рядкових значеннях). **Граничний випадок**: селектори в самому кінці файлу без наступного символу — пропускаються.
- **HTML/PHP файли (через cheerio)**:
  - ID: `$('#origin').attr('id', obfused)`
  - Класи: `$('.origin').removeClass(origin).addClass(obfused)`

#### 5. Запис результату

- Обфусковані CSS та HTML файли записуються в `opts.output + оригінальний_шлях`
- JSON-звіт записується в `opts.output + 'obscure.map'`

### `getDefinitions(cssString)` — CSS Парсер

Застосовує послідовний конвеєр очищення перед вилученням назв:

```js
var STRIP = {
  defs:     /\{[^\}]*?\}/g,          // видаляє тіла правил { ... }
  comments: /\/\*[\s\S]*?\*\//g,     // видаляє /* блочні коментарі */
  scope:    /\([\s\S]*?\)/g,         // видаляє (...) — умови медіа-запитів
  query:    /\[[\s\S]*?\]/g,         // видаляє [...] — атрибутні селектори
  exclude:  /<захардкоджений список>/g // видаляє конкретні підрядки
}
```

> **Виправлений баг**: `DEFINITIONS_CAP` — це глобальний regex (`/g`). До версії 1.1.0 він не скидав `lastIndex` між викликами `getDefinitions()`, що могло спричинити пропуск першого селектора у кожному наступному файлі. Тепер додано `DEFINITIONS_CAP.lastIndex = 0` на початку кожного виклику.

Після очищення використовує `/([#.][\w-]+)/g` для збору всіх токенів класів/ID.

> **Шаблон `STRIP.exclude` захардкоджений** і підігнаний під конкретний проєкт на Bootstrap. Він безумовно видаляє ці підрядки перед парсингом:
> ```
> collapse, show, social-icons, social-icon, links-social, links-text,
> navbar-nav, nav-item, nav-link, active, library, li:hover, font-marker,
> links-menu, navbar-toggler, navbar-toggler-icon, no-scroll, navbar, .open
> ```
> Будь-який проєкт, що використовує ці назви класів, мовчки пропустить їх обфускацію. Будь-яка назва класу, що **містить** один із цих підрядків (наприклад `.my-navbar-wrapper`), буде зламана під час вилучення.

### Формат обфускованої назви

```
r<результат_hashids>
```

- Префікс `r` (константа `SAFE_PREFIX`) гарантує, що назва завжди починається з літери (вимога валідності CSS/HTML).
- `hashids` ініціалізується із солью `"what does that say?"` та мінімальною довжиною виводу `5`.
- Приклад: сід `1700000000000`, індекс `0` → щось на кшталт `rK8mNp`.
- Детерміновано: однаковий сід + однаковий файл + однаковий порядок класів → однаковий результат при кожному запуску.
- Без `--seed` використовує `Date.now()` на старті процесу → різна обфускація при кожному запуску.

### Зміни у `safeWrite`

До версії 1.1.0 функція мала гонкову умову: `mkdirp` (async) + `fs.writeFileSync` (sync) всередині callback:
```js
// Старий небезпечний варіант
mkdirp(dir, (err) => {
    if (err) console.error(err);
    else fs.writeFileSync(fd, buffer); // міг виконатись до завершення mkdirp
});
```
Тепер обидві операції синхронні:
```js
fs.mkdirSync(dir, { recursive: true });
fs.writeFileSync(fd, buffer);
```

## Використання у Docker (як надано)

```dockerfile
FROM node:18-bullseye-slim as builder

RUN npm install -g obscure@1.0.1 --unsafe-perm

# Замінює index.js із npm-пакету на кастомний
COPY assets/obscure/index.js /usr/local/lib/node_modules/obscure/index.js

# Запуск обфускації
RUN node /usr/local/bin/obscure style.css --apply index.html --output /output/dist

# Копіювання необроблених ресурсів у вивід
RUN cp -vR EC2/ Apache/ SSL_certs/ assets/ ... /output/dist/

# Фінальний мінімальний образ
FROM alpine:3.18
COPY --from=builder /output/dist /dist
```

Ключові моменти:
- Глобальна npm-установка патчиться після інсталяції шляхом перезапису `index.js` кастомною версією з `assets/obscure/`.
- `--unsafe-perm` потрібен тому, що npm-установки під root у контейнері інакше пропускають lifecycle-скрипти (тут не критично, але безпечно залишити).
- Використовується багатоетапна збірка Docker; у фінальний Alpine-образ потрапляє лише `/output/dist`.

## Публікація в npm та Docker

### Публікація

1. **Змінити `name`** у `package.json` на свій scope, наприклад `@yourscope/obscure`

2. **Додати `.npmignore`** або поле `"files"`, щоб виключити зайве з пакету:
   ```json
   "files": ["index.js", "bin/", "README.md"]
   ```

3. **Винести список виключень** з `STRIP.exclude` в окремий конфіг-файл або додатковий CLI-прапор, щоб не залежати від Bootstrap.

4. Опублікувати:
   ```bash
   npm publish --access public
   ```

### Оновлений Dockerfile

Змінити рядок установки на нову версію та прибрати `--unsafe-perm` (не потрібен у v12 npm):

```dockerfile
RUN npm install -g @yourscope/obscure@1.1.0
# bin тепер коректно вказує на index.js — COPY більше не потрібен
```

Або якщо лишаєте старий підхід з патчем `index.js`:
```dockerfile
RUN npm install -g obscure@1.0.1
COPY assets/obscure/index.js /usr/local/lib/node_modules/obscure/index.js
```

### Відтворювані збірки (рекомендовано)

Додайте `--seed` у Docker `RUN` щоб обфускація була однаковою між збірками:
```dockerfile
RUN obscure style.css --apply index.html --output /output/dist --seed 42
```
